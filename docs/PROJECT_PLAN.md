# Project Plan & Change History

## 문서 목적
본 문서는 CellFusionC AI 추천 엔진의 실제 개발/운영 과정에서 발생한 주요 변경사항을 날짜 기준으로 기록합니다.  
기록 범위는 “무엇이 어떻게 바뀌었는지”, “왜 바꿨는지”, “어떤 개선이 있었는지”에 집중합니다.

## 프로젝트 개요
- 프로젝트명: CellFusionC AI 맞춤형 큐레이션 엔진 (MCP + GenUI)
- 목표: Cafe24 상품 데이터를 ChatGPT 내 네이티브 위젯으로 연결하고, 피부 타입/고민 기반 추천 품질과 운영 안정성을 동시에 확보
- 핵심 구성: Node.js(Express), MongoDB, MCP(JSON-RPC/SSE), React 위젯, 룰 기반 후보 선별 + LLM 재랭크

---

## 변경 이력

### 2026-04-01 ~ 2026-04-05: 기반 구축 및 실데이터 연동
**변경 내용**
- Cafe24 Admin API 연동 및 OAuth 인증 흐름 구축
- `search_cafe24_real_products` 초기 버전 구현
- MCP 서버 기본 구조 수립(SSE + JSON-RPC)

**변경 이유**
- 추천 엔진 고도화 이전에, 실데이터 기반 E2E 경로를 먼저 안정화할 필요가 있었음

**개선/영향**
- ChatGPT 도구 호출을 통해 실제 상품 데이터 반환 가능 상태 확보
- 운영 기초(토큰/동기화/툴 호출) 검증 완료

---

### 2026-04-06 ~ 2026-04-07: 태깅/스코어링 구조화
**변경 내용**
- `aiTaggingService` 도입(피부 타입/고민/제형/라인 태그 추출)
- 텍스트 포함 매칭 중심 로직을 속성 기반 가중치 점수 방식으로 전환
- 가격/이미지 필드 정규화 및 파싱 보정

**변경 이유**
- 초기 방식은 검색은 빠르지만 추천 정확도와 설명 가능성이 낮았음

**개선/영향**
- 피부 조건과 상충되는 후보가 상위에 노출되는 빈도 감소
- 추천 근거(점수 기반) 추적 가능성 향상

---

### 2026-04-08: 추천 응답 UX 보강
**변경 내용**
- 추천 문구를 나열형에서 상황형 코멘트 중심으로 개선
- 복합 피부 입력(예: 건성+민감성, 수부지) 처리 강화

**변경 이유**
- 추천 정확도 대비 사용자 체감 신뢰도가 낮다는 피드백 반영

**개선/영향**
- 사용자 질문 맥락에 맞는 설명 품질 향상
- 단일 타입 중심 추천 대비 실사용 적합도 개선

---

### 2026-04-09 ~ 2026-04-10: MCP 위젯 채택 안정화
**변경 내용**
- MCP 리소스 메타 정합성 정리
  - `_meta.ui.resourceUri`
  - `_meta["openai/outputTemplate"]`
  - `mimeType: text/html;profile=mcp-app`
- `/mcp`, `/mcp/sse`, `/mcp/message`, `/mcp/messages` 호환 경로 보강
- `resources/read` 템플릿 버저닝 도입
- 위젯 브리지의 메시지 언패킹 케이스 확장

**변경 이유**
- 세션/환경에 따라 위젯이 텍스트로 폴백되거나 로딩 상태에 머무는 문제 대응

**개선/영향**
- 위젯 채택 실패 원인 추적 가능성 상승
- 리소스 로드/툴 호출 흐름의 재현성 개선

---

### 2026-04-11 ~ 2026-04-13: 성능/안정성 패치 (운영 기준)
**변경 내용**
- 추천 파이프라인을 STAGE1(후보 선별) + STAGE2(재랭크) 운영 형태로 고정
- 캐시 준비 전 첫 요청 race condition 대응(on-demand sync 가드)
- 빈 추천 결과에서도 `structuredContent`/`_meta.widgetData`를 항상 반환하도록 보강
- 위젯에서 빈 결과 렌더 처리 추가(무한 로딩 방지)
- CORS/정적 리소스 서빙 보강

**변경 이유**
- 실서비스에서 지연/빈결과/무한 로딩이 사용자 이탈로 직결됨

**개선/영향**
- 첫 요청 실패율 감소
- 결과 없음 케이스의 UX 정상화
- 장애 시 fallback 경로 확보

---

### 2026-04-13: 운영 로그 정책/문서 정비
**변경 내용**
- 공통 로거(`utils/logger.js`) 도입
- 로그 플래그 분리
  - `LOG_LEVEL`
  - `LOG_MCP_VERBOSE`
  - `LOG_CACHE_FILTER`
  - `LOG_TOKEN_EVENTS`
- Mongoose 경고 제거
  - `new: true` → `returnDocument: 'after'`
- 운영 문서 정리(`README`, `docs/LOGGING.md`, 본 문서)

**변경 이유**
- 고빈도 로그로 인해 실제 장애 포인트 탐지 시간이 길었음

**개선/영향**
- 운영 로그 가독성 향상
- 디버깅 모드와 운영 모드 분리 가능
- 문서 기반 온보딩 효율 개선

---

## 현재 상태 요약
- MCP 연결/리소스 로드/툴 호출 흐름은 운영 가능한 수준으로 안정화됨
- 추천 품질은 룰 기반 단일 구조에서 하이브리드 구조로 고도화됨
- 남은 과제는 “정확도 유지 + 다양성 개선 + 위젯 UX 밀도 최적화”

2. 프로모션 가중치 정책 정교화(기본상품/행사상품 균형)
3. 위젯 카드 정보 밀도 표준화(가격/핵심포인트/사용팁/주의사항)
